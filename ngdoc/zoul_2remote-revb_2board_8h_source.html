181 /**
182  * \name ADC configuration
183  *
184  * These values configure which CC2538 pins and ADC channels to use for the ADC
185  * inputs. By default the RE-Mote allows two out-of-the-box ADC ports with a
186  * phidget-like 3-pin connector (GND/VDD/ADC)
187  *
188  * The RE-Mote allows both 3.3V and 5V analogue sensors as follow:
189  *
190  * - ADC1 (PA5): up to 3.3V.
191  * - ADC2 (PA4): up to 3.3V
192  * - ADC3 (PA2): up to 5V, by means of a 2/3 voltage divider.
193  *
194  * Also there are other ADC channels shared by default with Micro SD card and
195  * user button implementations:
196  *
197  * - ADC4 (PA6): up to 3.3V.
198  * - ADC5 (PA7): up to 3.3V.
199  * - ADC6 (PA3): up to 3.3V.
200  *
201  * ADC inputs can only be on port A.
202  * All ADCx are exposed in JP5 connector, but only ADC1 and ADC3 have GND and
203  * VDD (3/5V) pins next to it, so these can be exposed into a 3-pin phidget-like
204  * connector, for ADC2 either solder a wire to connect, or use a 4-pin connector
205  * to expose both ADC1 and ADC2 in a single connector, but this will leave no
206  * space for a ADC3 connector.
207  *
208  * The internal ADC reference is 1190mV, use either a voltage divider as input,
209  * or a different voltage reference, like AVDD5, or externally using PA7/AIN7
210  * and PA6/AIN6 configurable as differential reference, by removing the R26 and
211  * R33 0Ohm resistors to disconnect off the Micro-SD, and those will be
212  * accessible from JP5 connector.
213  *
214  * To enable the ADC[2,4-6], remove any 0Ohm resistors if required (see above),
215  * and define in your application `ADC_SENSORS_CONF_ADCx_PIN` and set its
216  * value with the corresponding pin number (i.e ADC2 to 4 as mapped to PA4).
217  * To disable any ADC[1-6] just define as above, but set to (-1) instead.
218 
219  * Warning: if using ADC6 (PA3), you will need to disable the bootloader by
220  * making FLASH_CCA_CONF_BOOTLDR_BACKDOOR equal to zero
221  *
222  * @{
223  */
224 #define ADC_SENSORS_PORT GPIO_A_NUM /**< ADC GPIO control port */
225

277 /** \name RE-Mote Button configuration
278  *
279  * Buttons on the RE-Mote are connected as follows:
280  * - BUTTON_USER -> PA3, S1 user button, shared with bootloader
281  * - BUTTON_RESET -> RESET_N line, S2 reset the CC2538
282  * - BUTTON_PWR -> Depending on the enabled resistor, it can be used to reset
283  * the onboard Low-power PIC, provoking a master reset on all
284  * the RE-Mote's onboards components. Note the BUTTON_RESET
285  * only resets the CC2538. This is disabled by default, as
286  * the R45 0Ohm resistor is not soldered on that position
287  * The other R45 position enables a test-button to drive the
288  * SYSOFF pin of the power management block, disconnecting the
289  * battery when used, leaving only powered the RTCC and
290  * Low-Power PIC. Useful if developing applications using the
291  * shutdown mode if required to snap out of it.
292  * @{
293  */

449 /**
450  * \name Power management and shutdown mode
451  *
452  * The shutdown mode is an ultra-low power operation mode that effectively
453  * powers-down the entire RE-Mote (CC2538, CC1200, attached sensors, etc) and
454  * only keeps running a the on-board RTC and an ultra-low power consumption MCU
455  * The Shutdown mode allows:
456  *
457  * - Put the board in an ultra-low power sleep (shutdown) drawing <150nA avg.
458  * - Awake the system by scheduling the RTCC to awake the Low-Power PIC after
459  * it disconnects the battery and goes to sleep mode.
460  * - Awake the system by using the Low-Power PIC's timer
461  *
462  * As commented above, S3 can be used to restart the entire board (power
463  * management block included), or to kick the board out of shutdown mode by
464  * reconnecting the battery.
465  * @{
466  */

468 #define PM_ENABLE_PIN 1

471 /**
472  * \name On-board RTCC
473  *
474  * The on-board RTCC (real time clock-calendar) is powered over USB/battery,
475  * and it will remain powered in shutdown mode with the Low-Power PIC. The
476  * RTC_INT1 is connected to the CC2538, so it is possible to receive interrupts
477  * from a pre-configured alarm, even waking up the CC2538 from PM3.
478  * A second interruption pin (RTC_INT2) is connected to the Low-Power PIC, after
479  * configuring the RTCC the Low-Power PIC can drive the board to shutdown mode,
480  * and enter into low-power mode (sleep), being the RTCC interrupt the waking up
481  * source to resume operation.
482  *
483  * @{
